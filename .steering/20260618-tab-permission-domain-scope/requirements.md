# 要求内容

## 概要

レベルB の「進む」一時許可を、機能設計書どおり「同一タブ内で別ドメインへ遷移したら破棄」する挙動に揃える。現状はタブを閉じる（または SW 揮発）まで許可が残るため、設計より緩い。

## 背景

ステップ6でレベルB の再発動防止（タブ一時許可マップ）を実装したが、`chrome.tabs.onRemoved`（タブ閉じ）と SW 揮発による失効のみで、機能設計書が定める「ドメインが変わったら破棄」が未実装。そのため、許可済みドメインから別ドメインへ移動して戻っても再確認されず、許可が意図より長く継続する。

機能設計書（`docs/functional-design.md`）の該当記述:
> `chrome.tabs.onRemoved` でタブが閉じられたら、またはドメインが変わったらマップから削除。

## 実装対象の機能

### 1. ドメイン変更時の一時許可破棄
- 同一タブで、許可済みドメイン（およびそのサブドメイン）の範囲外へメインフレーム遷移したとき、そのタブの一時許可を破棄する
- 破棄後、許可済みだったサイトへ再訪すると再びレベルB 確認（またはC リダイレクト）が発動する

## 受け入れ条件

### ドメイン変更時の破棄
- [ ] レベルB で youtube を「進む」許可 → 同一タブで別ドメイン（例 google.com）へ遷移 → 再度 youtube へ遷移すると**再確認される**
- [ ] 許可済みドメインのサブドメイン間遷移（例 `youtube.com` ↔ `m.youtube.com`）では許可が維持される（破棄しない）
- [ ] 同一ドメイン内のページ遷移では許可が維持され再発動しない（既存挙動の非回帰）

### 既存機能の非回帰
- [ ] タブを閉じたら許可破棄（既存）は維持
- [ ] レベルC リダイレクト・全体OFF / 個別OFF / 対象外素通しは不変

### 品質
- [ ] `chrome-extension` の type-check / lint がエラー0
- [ ] `pnpm build` 成功

## スコープ外
- SPA 内 pushState 遷移への追従（MVP はフルページ遷移のみ）
- 許可の TTL（時間期限）導入
