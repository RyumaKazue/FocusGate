# 要求内容

## 概要

MVP 実装計画ステップ6。警告レベルB（確認オーバーレイ）を content-ui の Shadow DOM で実装し、対象サイトを開いた際に「進む / やめる」の確認を挟む。加えて、同一タブ・同一ドメインで一度「進む」を選んだら再遷移では再発動しない「一時許可」機構を SW に実装する。

## 背景

ステップ5までで全 UI と縦スライス（レベルC=完全ブロック）が完成済み。レベルB は「即ブロックではなく、ユーザーに一拍置かせて意図を確認する」緩やかな関所であり、FocusGate の中核体験。content script は描画完了後に注入されるため、レベルB は `onCompleted` で発火し、content-ui のオーバーレイをメッセージで起動する。

## 実装対象の機能

### 1. レベルB 確認オーバーレイ（content-ui）
- 対象サイトをレベルB で開くと、画面に確認オーバーレイ（Shadow DOM）が表示される
- 対象サイト名（label 優先）と「進む」「やめる」ボタンを表示
- 「進む」→ オーバーレイ解除＋当該タブ・ドメインを一時許可（SW へ通知）
- 「やめる」→ オーバーレイ解除＋ `history.back()` で離脱

### 2. 再発動防止（SW の一時許可機構）
- 「進む」で許可したタブ＋ドメインは、同一タブでの再遷移で再確認しない（B/C とも素通し）
- インメモリ `Map<number, Set<string>>`（tabId → 許可ドメイン）で管理
- タブを閉じたら（`onRemoved`）当該タブの許可を破棄 → 別タブ／タブ閉じ後は再確認

### 3. SW のレベルB 発火（onCompleted）
- `chrome.webNavigation.onCompleted`（メインフレーム）で判定し、`blocked && level==='B'` かつ未許可なら content へ `SHOW_WARNING` を送る
- レベルC は従来どおり `onBeforeNavigate` でリダイレクト（変更なし）

## 受け入れ条件

### レベルB オーバーレイ
- [ ] レベルB 設定で対象サイトを開くと確認オーバーレイが表示される
- [ ] オーバーレイに対象サイト名（label ?? domain）が表示される
- [ ] 「やめる」で前ページへ離脱する
- [ ] 「進む」でオーバーレイが消えサイトを閲覧できる

### 再発動防止
- [ ] 「進む」後、同一タブでの当該サイト再遷移では再発動しない
- [ ] 別タブで開くと再度確認される
- [ ] タブを閉じて開き直すと再度確認される

### 既存機能の非回帰
- [ ] レベルC は引き続き blocked.html へリダイレクトされる
- [ ] 全体OFF / 個別OFF / 対象外サイトは素通し

### 品質
- [ ] `pnpm build` が成功し `dist/content-ui/focusgate.iife.js` が生成される
- [ ] lint エラー0、編集ファイル由来の型エラー0

## 成功指標

- レベルB の「一拍置く」関所体験が成立し、進んだ後は同一タブで邪魔されない
- レベルB/C の切替が設定どおりに挙動へ反映される

## スコープ外

- SPA 内遷移（pushState 等）への追従（MVP はフルページ遷移のみ）
- オーバーレイのデザイン作り込み（刺激の少ない最小トーン）
- E2E テスト（ステップ7）
