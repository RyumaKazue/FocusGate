# 要求内容

## 概要

拡張機能のポップアップUIの表示サイズを現在の 300x260px から 400x360px に拡大する。

## 背景

現在のポップアップは横幅300px・高さ260pxと小さく、表示できる情報量や操作領域に余裕がない。サイズを拡大することで、UIの視認性と操作性を改善する。

## 実装対象の機能

### 1. ポップアップサイズの拡大
- `pages/popup/src/index.css` の `body` の `width`/`height` を変更する
- 横幅: 300px → 400px
- 高さ: 260px → 360px

### 2. UIをポップアップサイズに追従させる（追加対応）
- サイズ拡大後、UIコンテンツがbody（360px）の高さに満たず、下部に空白が生じている
- ルート要素（`#app-container` および Popup ルートdiv）を body の高さいっぱいに広げ、空白をなくす
- サイトリスト領域が余白を吸収し、操作ボタンが下部に配置されるようにする

## 受け入れ条件

### ポップアップサイズの拡大
- [ ] `pages/popup/src/index.css` の `body { width: 400px; height: 360px; }` になっている
- [ ] ビルドが成功する（`pnpm build`）
- [ ] 型チェックが通る（`pnpm type-check`）
- [ ] リントが通る（`pnpm lint`）

### UIをポップアップサイズに追従させる
- [ ] ポップアップ表示時に下部の空白がなくなり、UIがbody全体（400x360px）を占める
- [ ] 背景色（bg-slate-50）が下部まで途切れずに表示される
- [ ] サイト数が少なくてもレイアウトが崩れず、操作ボタンが下部に収まる

## 成功指標

- ポップアップが 400x360px で表示され、UIが全体を隙間なく占める

## スコープ外

以下はこのフェーズでは実装しません:

- ポップアップ内部のコンテンツ（項目）自体の追加・再設計
- レスポンシブ対応やサイズの動的変更
