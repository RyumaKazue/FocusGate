# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**FocusGate** - 集中を妨げるサイトへのアクセスを「関所」で制御する集中支援 Chrome 拡張機能

### プロダクトコンセプト
- **無意識の行動への介入**: YouTube や SNS を「気づいたら開いていた」という無意識のアクセスに対し、その瞬間に気づき／抑止を与える。
- **段階的な抑止レベル**: 「確認ワンクッション」「完全ブロック」の2段階から、自分に合った強さを選べる。
- **ローカル完結・軽量**: 設定はすべてローカル（`chrome.storage.local`）に保存。外部サーバー不要で、プライバシーを保ちつつ軽快に動作する。

### プロダクトビジョン
FocusGate は、勉強や開発などに集中したい人が、無意識のうちに集中を妨げるサイトを開いてしまう「癖」を矯正するための Chrome 拡張機能である。
集中を妨げるサイトへのアクセスを「関所（Gate）」で制御し、アクセスのタイミングで気づきと抑止を与える。
強制的な遮断だけでなく、ユーザー自身が抑止の強さを選べる段階的な仕組みにより、無理なく行動変容を促す。
設定はローカルに保存され、シンプルなポップアップと詳細設定用のオプションページで、誰でも直感的に操作できる。

### 目的
- 無意識に集中阻害サイトを開いてしまう行動に「気づき」を与える。
- 段階的な抑止により、ユーザー自身のペースで集中を妨げる癖を矯正する。
- 設定変更や ON/OFF 切替を最小限の操作で完結させ、ツール自体が集中の妨げにならないようにする。

## ターゲットユーザー

### プライマリーペルソナ: 佐藤健太(20歳、大学生)
- 試験勉強やレポート作成のため、自室の PC で長時間作業する。
- 作業中、無意識のうちに新しいタブを開いて YouTube を見てしまい、気づくと30分経っている。
- 「見ないようにしよう」という意志だけでは止められず、物理的・仕組み的な抑止が欲しい。
- IT に特別詳しいわけではないため、インストール後すぐ使えるシンプルさを求める。
- **1日の典型的なワークフロー**: 朝〜夜にかけて断続的に PC で学習。集中したい時間帯と休憩時間が混在する。

### セカンダリーペルソナ: 鈴木美咲(28歳、ソフトウェアエンジニア)
- 在宅勤務で1日中ブラウザと IDE を使って開発する。
- 詰まったときの気晴らしに SNS を開く癖があり、そのまま長時間離脱してしまう。
- ブロックの強さを時と場合で変えたい(集中したいときは完全ブロック、休憩中は OFF)。
- Chrome 拡張の操作には慣れており、ツールバーから素早く切り替えたい。
- **1日の典型的なワークフロー**: 業務時間中は集中モード、休憩や昼休みは OFF にして自由に閲覧。

## 成功指標(KPI)

### プライマリーKPI
- ブロック対象サイトへのアクセス試行に対するブロック発動成功率: 99%以上(対象サイトを開いたとき確実に発動する)
- 設定変更からブロック挙動への反映時間: 即時(次回アクセス時に必ず反映)
- 拡張機能の主要操作(全体 ON/OFF、警告レベル変更)の完了までの操作ステップ: 3ステップ以内

### セカンダリーKPI
- 初回インストールから初期設定完了までの時間: 3分以内(初期ブロックリストが登録済みのため、追加設定なしで使い始められる)
- カスタムサイト1件の追加完了までの操作ステップ: 3ステップ以内
- 拡張機能起動(ポップアップ表示)からの応答時間: 200ms 以内

## 機能要件

### コア機能(MVP)

#### 1. ブロック対象サイトの管理

**ユーザーストーリー**:
集中したいユーザーとして、自分が誘惑されやすいサイトを抑止対象にするために、ブロック対象サイトを管理できる機能が欲しい。

**受け入れ条件**:
- [ ] 初回インストール時、初期ブロックリストとして YouTube、TikTok、Instagram、Facebook が登録済みである。
- [ ] ユーザーが任意のドメインをブロック対象として追加できる。
- [ ] 登録済みのブロック対象サイトを編集できる。
- [ ] 登録済みのブロック対象サイトを削除できる。
- [ ] ブロック単位はドメイン全体である(例: `youtube.com` 配下のすべてのページが対象)。
- [ ] サブドメイン(例: `m.youtube.com`)も同一ドメインのブロック対象として扱われる。
- [ ] 不正な形式のドメイン入力時はエラーを表示し、登録を拒否する。

**優先度**: P0(必須)

#### 2. 警告／ブロックの発動

**ユーザーストーリー**:
集中したいユーザーとして、無意識に開いたサイトで集中を取り戻すために、ブロック対象サイトを開いたときに警告／ブロックが発動して欲しい。

**受け入れ条件**:
- [ ] 「全体フラグが ON」かつ「対象サービスのフラグが ON」の場合にのみブロックが発動する。
- [ ] 全体フラグが OFF の場合、いかなる対象サイトでも発動しない。
- [ ] 対象サービスのフラグが OFF の場合、そのサイトでは発動しない。
- [ ] ブロック対象外のサイトでは一切発動せず、通常通り閲覧できる。
- [ ] 発動時の挙動は、設定された警告レベル(B/C)に従う。

**優先度**: P0(必須)

#### 3. 警告レベルの選択(全体共通設定)

**ユーザーストーリー**:
集中したいユーザーとして、自分に合った抑止の強さを選ぶために、警告レベルを段階的に選択できる機能が欲しい。

**受け入れ条件**:
- [ ] 警告レベルはプロダクト全体で1つの共通設定である(サービス個別ではない)。
- [ ] レベル B(確認ワンクッション): 「本当に開きますか?」と確認し、ユーザーがボタンを押せばページに進める。
- [ ] レベル C(完全ブロック): ページを表示させず、別画面(ブロック画面)に置き換える。
- [ ] 警告レベルの変更後、次回のアクセスから新しいレベルが適用される。
- [ ] デフォルトの警告レベルが定義されている(レベル B を推奨デフォルトとする)。

> **補足**: 当初は「レベル A(警告のみ・閲覧継続)」を含む3段階だったが、A は抑止効果が弱く無視されやすいため廃止し、**B(確認)と C(完全ブロック)の2段階**とする。

**優先度**: P0(必須)

#### 4. ON/OFF 制御

**ユーザーストーリー**:
集中したいユーザーとして、集中したいときと休憩したいときを切り替えるために、ブロックの ON/OFF を制御できる機能が欲しい。

**受け入れ条件**:
- [ ] 全体の ON/OFF を切り替えられる。
- [ ] サービス(登録サイト)ごとに ON/OFF を切り替えられる。
- [ ] 全体 OFF のとき、サービス個別の設定にかかわらずブロックは発動しない。
- [ ] ON/OFF の状態が UI 上で視覚的に判別できる。
- [ ] 切替後の状態が即座に保存され、次回アクセスに反映される。

**優先度**: P0(必須)

#### 5. 設定の保存

**ユーザーストーリー**:
集中したいユーザーとして、ブラウザを再起動しても設定を維持するために、設定がローカルに保存される機能が欲しい。

**受け入れ条件**:
- [ ] ブロックリスト、各種 ON/OFF、警告レベルがローカル(`chrome.storage.local`)に保存される。
- [ ] ブラウザ／PC を再起動しても設定が保持される。
- [ ] 設定は外部サーバーに送信されない(ローカルのみ)。
- [ ] 初回起動時、保存済み設定がなければ初期値(初期ブロックリスト・デフォルト警告レベル・全体 ON)で初期化される。

**優先度**: P0(必須)

#### 6. 操作 UI(ポップアップ・オプションページ)

**ユーザーストーリー**:
集中したいユーザーとして、素早く設定を切り替えたり詳細を編集したりするために、ポップアップとオプションページの両方が欲しい。

**受け入れ条件**:
- [ ] ツールバーアイコンからポップアップを開ける。
- [ ] ポップアップで全体 ON/OFF を切り替えられる。
- [ ] ポップアップで警告レベル(B/C)を切り替えられる。
- [ ] ポップアップで登録済みサービスごとの ON/OFF を切り替えられる。
- [ ] オプションページでブロックリストの追加・編集・削除ができる。
- [ ] ポップアップとオプションページの設定変更が相互に整合する(一方の変更が他方に反映される)。

**優先度**: P0(必須)

### 将来的な機能(Post-MVP)

#### スケジュール機能
「平日 9〜18時だけブロック」など、時間帯による自動 ON/OFF を行う。

**優先度**: P2(できれば)

#### 設定の同期
`chrome.storage.sync` などを利用し、複数 PC / ブラウザ間で設定を同期する。

**優先度**: P2(できれば)

#### パス単位のブロック
「YouTube は見たいが Shorts だけブロック」など、ドメインより細かい URL/パス単位の制御を行う。

**優先度**: P2(できれば)

#### ブロック回数の記録・可視化
「今日 YouTube を○回開こうとした」など、無意識のアクセス試行回数を記録し、振り返れるようにする。

**優先度**: P2(できれば)

## 非機能要件

### パフォーマンス
- ブロック判定処理: ページ遷移時の判定を 50ms 以内に完了し、体感的な遅延を発生させない。
- ポップアップ表示: ツールバーアイコンクリックから 200ms 以内に表示する。
- ブラウジングへの影響: ブロック対象外サイトの読み込み速度に体感できる影響を与えない。

### ユーザビリティ
- 新規ユーザーがインストール後、追加設定なしで(初期ブロックリストにより)すぐに利用を開始できる。
- 主要操作(全体 ON/OFF、警告レベル変更)を3ステップ以内で完了できる。
- ON/OFF や警告レベルの現在状態が UI 上で一目で判別できる。

### 信頼性
- 設定変更時にデータ損失が発生しない(保存失敗時はエラーを通知)。
- ブラウザ再起動後も設定が確実に復元される。
- ブロック判定が「全体フラグ ON かつ サービスフラグ ON」のときのみ発動し、誤作動(対象外サイトのブロック等)が起きない。

### セキュリティ・プライバシー
- ユーザーの閲覧データ・設定を外部に送信しない(ローカル完結)。
- 必要最小限の Chrome 権限のみを要求する(ブロック対象サイト・ストレージに関わる権限など)。
- 拡張機能が収集する個人情報はない。

### 互換性
- Google Chrome の Manifest V3 に準拠する。
- 最新版およびその1つ前のメジャーバージョンの Chrome で動作する。

## スコープ外

現時点(MVP)で明示的にスコープ外とする項目:
- スケジュール／時間帯による自動 ON/OFF 制御(将来機能)
- クラウド同期・複数端末間の設定同期(将来機能)
- パス・URL 単位の細かいブロック(将来機能)
- ブロック試行回数の統計・レポート・可視化(将来機能)
- Chrome 以外のブラウザ(Firefox、Edge、Safari 等)への対応
- 警告レベルのサービス個別設定(MVP では全体共通のみ)
