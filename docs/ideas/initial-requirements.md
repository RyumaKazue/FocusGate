# FocusGate 初期要求仕様（壁打ちメモ）

> このドキュメントは PRD 作成前の壁打ちメモです。プロダクトの基本アイデア・課題・MVP の範囲を整理したものです。

## プロダクトの基本アイデア

- **名称**: FocusGate
- **形態**: Chrome 拡張機能
- **一言で**: ブロックリストに登録したサイト（YouTube, TikTok など）を開こうとすると、警告またはブロックを行う集中支援ツール。
- **名前の由来**: Focus（集中）+ Gate（門・関所）。集中を妨げるサイトへのアクセスを「関所」で制御するイメージ。

## 解決したい課題

- 勉強・開発など作業に集中したいのに、**無意識のうちに YouTube や SNS を開いてしまう癖**がある。
- 「気づいたら見ていた」という無意識の行動に対して、アクセスのタイミングで気づき／抑止を与え、**癖そのものを矯正する**ことを目指す。

## ターゲットユーザー

- **勉強や開発など、作業に集中したい人**全般。
- 集中したいのに無意識に SNS / 動画サイトを開いてしまう人。
- 想定例: 学習中の学生、集中して作業したい開発者・社会人。

## 主要機能（MVP の範囲）

### 1. ブロック対象サイトの管理
- 初期ブロックリストとして以下を登録済みにする:
  - YouTube
  - TikTok
  - Instagram
  - Facebook
- ユーザーがブロック対象サイトを **カスタムで追加・編集・削除**できる。
- ブロック単位は **ドメイン全体**（例: `youtube.com` 配下すべて）。

### 2. 警告／ブロックの発動
- ブロック対象サイトを開こうとすると警告／ブロックが発動する。
- **発動条件**: 「全体フラグが ON」**かつ**「対象サービスのフラグが ON」の場合にブロック対象となる。

### 3. 警告レベルの選択（全体で1つの設定）
ユーザーは以下の3段階から警告の強さを選択できる。設定は**プロダクト全体で共通（サービス個別ではない）**。
- **レベル A（警告のみ）**: 「集中中です」といった通知を表示するが、そのまま閲覧は可能（気づきを与える）。
- **レベル B（確認ワンクッション）**: 「本当に開きますか？」と確認し、ボタンを押せば進める（衝動に一拍置く）。
- **レベル C（完全ブロック）**: ページを表示させず、別画面に置き換える（物理的に遮断）。

### 4. ON/OFF 制御
- **全体の ON / OFF** が可能。
- **サービスごとの ON / OFF** が可能。

### 5. 設定の保存
- 設定（ブロックリスト、各種 ON/OFF、警告レベル）は **ローカルのみに保存**（`chrome.storage.local` 想定）。

### 6. 操作 UI
- **ポップアップ**: ツールバーアイコンから開く。全体 ON/OFF や警告レベルの切替などをサッと操作。
- **オプションページ**: ブロックリストの追加・編集・削除など詳細設定を行う。
- MVP では**ポップアップとオプションページの両方**を実装する。

## 将来的な機能（Post-MVP）

以下は MVP には含めず、後から機能追加として実装予定。

- **スケジュール機能**: 「平日 9〜18時だけブロック」など時間帯による自動 ON/OFF。
- **設定の同期**: `chrome.storage.sync` などで複数 PC / ブラウザに設定を同期。
- **パス単位のブロック**: 「YouTube は見たいが Shorts だけブロック」など、ドメインより細かい URL/パス単位の制御。
- **ブロック回数の記録・可視化**: 「今日 YouTube を○回開こうとした」など、無意識のアクセス試行回数を記録して振り返れるようにする。

## スコープ外（現時点）

- スケジュール／時間帯制御（将来機能）
- クラウド同期・複数端末同期（将来機能）
- パス・URL 単位の細かいブロック（将来機能）
- ブロック試行回数の統計・レポート（将来機能）
